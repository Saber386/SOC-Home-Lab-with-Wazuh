# Wazuh Architecture

## Overview

Wazuh follows a centralized architecture where security events generated by monitored endpoints are collected, processed, indexed, and visualized through a web-based dashboard. It enables centralized security monitoring, log analysis, threat detection, and incident investigation across multiple systems.

---

## Architecture Components

### Wazuh Agent

The Wazuh Agent is installed on monitored endpoints such as Windows, Linux, or macOS systems. It collects security events, system logs, file integrity information, vulnerability data, and other endpoint telemetry before securely forwarding it to the Wazuh Server.

**Functions:**

* Log Collection
* File Integrity Monitoring (FIM)
* Vulnerability Detection
* System Inventory
* Security Configuration Assessment

---

### Wazuh Server

The Wazuh Server acts as the central management component. It receives events from agents, applies detection rules, correlates events, and generates security alerts.

**Functions:**

* Agent Management
* Rule Processing
* Event Correlation
* Alert Generation
* Security Monitoring

---

### Wazuh Indexer

The Wazuh Indexer stores security events and alerts received from the server. It provides fast indexing and efficient searching of security data.

**Functions:**

* Event Storage
* Data Indexing
* Fast Search
* Historical Log Retention

---

### Wazuh Dashboard

The Wazuh Dashboard provides a graphical interface for monitoring security events, viewing alerts, managing agents, and analyzing collected data.

**Functions:**

* Dashboard Visualization
* Alert Monitoring
* Agent Management
* Log Search
* Security Reporting

---

## Architecture Workflow

```text
Windows / Linux Endpoint
          │
          ▼
      Wazuh Agent
          │
          ▼
      Wazuh Server
          │
          ▼
     Wazuh Indexer
          │
          ▼
    Wazuh Dashboard
          │
          ▼
   Security Analyst
```

---

## Data Flow

1. Security events are generated on monitored endpoints.
2. The Wazuh Agent collects the events.
3. Events are securely transmitted to the Wazuh Server.
4. The server analyzes events using predefined rules.
5. Security alerts are generated when suspicious activity is detected.
6. Events and alerts are indexed by the Wazuh Indexer.
7. Analysts investigate alerts through the Wazuh Dashboard.

## Conclusion

The Wazuh architecture provides a scalable and centralized platform for security monitoring by combining endpoint agents, a management server, an indexing engine, and a web-based dashboard. Understanding the interaction between these components is essential before deploying and configuring a Wazuh-based SIEM environment.
